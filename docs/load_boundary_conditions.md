# 荷重条件・境界条件 — 本プロジェクト向け設計指針

> H3 フェアリング CFRP/Al-Honeycomb GNN-SHM プロジェクト
> デボンディング欠陥検出に適した FEM 荷重・境界条件の整理

---

## 1. 現状の実装

### 1.1 境界条件 (BC)

| 項目 | 現行値 | 実装箇所 |
|------|--------|----------|
| **固定端** | y=0 (バレル基部) 全自由度固定 | `apply_boundary_conditions()` |
| **拘束** | u1=0, u2=0, u3=0 (DisplacementBC) | `generate_realistic_fairing.py` L639-641 |
| **対象** | 内板・外板エッジ + コア底面 | `BC_Bottom` Set |
| **対称** | 1/6 セクション (θ=60°弧) | 幾何で暗黙的にモデル化 |

**物理的根拠**: フェアリング基部はロケット本体にボルト固定（ノッチドボルト列）。分離前は z=0 を固定端とみなす。

### 1.2 荷重条件

| 荷重種別 | 現行 | JAXA文献値 | 備考 |
|----------|------|-------------|------|
| **熱荷重** | ✅ 適用 | 外板 100–200℃, 内板 ~50℃ | 主荷重 |
| **空力圧力** | ❌ 未適用 | Max Q 30–40 kPa | 検証用 50 kPa は FEniCS のみ |
| **音響** | ❌ 未適用 | ~148 dB | SHM 主目標だが FEM では省略 |
| **分離衝撃** | ❌ 未適用 | ~1,500 G | 分離解析用 |

**現行の熱荷重パラメータ** (`generate_realistic_fairing.py`, `patch_inp_thermal.py`):

```python
TEMP_INITIAL = 20.0   # ℃  基準（応力フリー）
TEMP_FINAL_OUTER = 120.0   # 外板（均一）または z 依存 100–220℃
TEMP_FINAL_INNER = 20.0   # 内板
TEMP_FINAL_CORE  = 70.0   # コア
```

## 3. 本プロジェクトに適した設計指針

### 3.1 主目的との整合

| 目的 | 荷重の優先度 | 理由 |
|------|--------------|------|
| **デボンディング検出** | 熱荷重 >> 空力圧力 | CTE 不整合が界面剥離の主因 |
| **GNN 特徴量** | 変位・応力・温度 | 熱応力が欠陥シグナルを増幅 |
| **SHM 学習** | 定常ピーク条件で十分 | 非定常解析はコスト過大 |

### 3.2 推奨境界条件

| 項目 | 推奨 | 根拠 |
|------|------|------|
| **固定端** | y=0 全自由度固定 | 現行のまま。分離前の実機に相当 |
| **対称** | 1/6 セクション (θ=60°) | 計算効率。θ=0°/60° で Uθ=0 を厳密に課す場合は将来検討 |
| **リングフレーム** | Tie 拘束で内板に接合 | 現行実装済み |

### 3.3 推奨荷重条件

| 荷重 | 推奨 | 根拠 |
|------|------|------|
| **熱荷重** | **必須** | デボンディング検出の主駆動力。外板 100–150℃, 内板 20–50℃ |
| **z 依存温度** | **推奨** | `patch_inp_thermal.py` のオジャイブ加熱プロファイル（基部 100℃→ノーズ 220℃）が物理的に妥当 |
| **空力圧力** | オプション | Max Q 30–50 kPa を外板に追加すると応力・変位が増幅。熱のみでも学習は可能 |
| **音響** | 省略 | リニア静解析では扱いにくい。SHM は熱+構造応答で代用 |

### 3.4 熱荷重パラメータの推奨範囲

| パラメータ | 最小 | 推奨 | 最大 | 根拠 |
|------------|------|------|------|------|
| T_OUTER | 100℃ | 120–150℃ | 200℃ | JAXA 外板 100–200℃ |
| T_INNER | 20℃ | 20–50℃ | 50℃ | 内板遮蔽 |
| T_CORE | 50℃ | 70℃ | 90℃ | 外・内の線形補間 |
| T_REF | 20℃ | 25℃ | 25℃ | 応力フリー基準 |

## 4. 将来拡張案

### 4.1 空力圧力の追加

Abaqus で外板外面に均一圧力 30–50 kPa を `*Dsload` で適用する場合:

```python
# 外板外面サーフェスに圧力 (Abaqus)
model.Pressure(name='Pressure_MaxQ', createStepName='Step-1',
               region=surf_outer_external, distributionType=UNIFORM,
               magnitude=-50.0)  # kPa, 外側向きは負
```

- **利点**: 変位・応力が増大し、欠陥シグナルが明確になる可能性
- **注意**: 熱+圧力の複合荷重となり、ODB 抽出・特徴量解釈が複雑化

### 4.2 対称境界の厳密化

1/6 セクションで θ=0°, 60° に `UR2=0` (周方向回転固定) を課すと、円筒対称性がより正確に再現される。現行は幾何カットのみで暗黙的。

### 4.3 音響荷重の簡易モデル

RMS 圧力 ~5 kPa を擬似静圧として外板に重畳する方法があるが、周波数依存性を無視するため SHM 学習への寄与は限定的。

## 5. まとめ

| カテゴリ | 現行 | 推奨 | 備考 |
|----------|------|------|------|
| **境界条件** | y=0 固定 | 現行維持 | 妥当 |
| **熱荷重** | 外 120℃, 内 20℃ | z 依存プロファイル推奨 | 空力加熱を反映 |
| **空力圧力** | 未適用 | オプション (30–50 kPa) | 検証・感度解析向け |
| **音響** | 未適用 | 省略 | 静解析の範囲外 |

**結論**: デボンディング検出を主目的とする GNN-SHM では、**熱荷重を主荷重とし、境界条件は現行の y=0 固定を維持**するのが適切。空力圧力は感度解析や検証時に追加検討可能。
