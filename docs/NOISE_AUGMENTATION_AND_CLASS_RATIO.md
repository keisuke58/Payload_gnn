# ノイズ拡張とクラス比率の提案

実測赤外線応力測定のノイズをシミュレーションに乗せ、データ量向上と実運用に近いクラス比率を検討する。

---

## 1. ノイズ拡張の意義

### 1.1 実測 IR のノイズ要因

| 要因 | 説明 | 典型的な影響 |
|------|------|--------------|
| **センサノイズ** | 検出器の熱雑音・読み出しノイズ | SNR 比 40–60 dB |
| **放射率むら** | 表面の反射率・放射率の局所変動 | 温度換算で ±1–3 K |
| **環境変動** | 周囲温度・風・日射 | ドリフト・低周波成分 |
| **空間分解能** | ピクセルサイズ・ぼけ | 高周波成分の減衰 |
| **ロックイン位相誤差** | 位相検出の揺らぎ | 応力換算で数 % |

### 1.2 シミュレーションへの適用

- **FEM 出力はノイズフリー** → 実測との domain gap が大きい
- **ノイズ付加**により：
  - 実測に近い分布で学習できる
  - 過学習を抑制
  - データ拡張として有効サンプル数を増やせる

### 1.3 ノイズ付加対象

`build_graph.py` のノード特徴量のうち、測定由来のもの：

| 特徴 | 次元 | ノイズ付加 | 理由 |
|------|------|------------|------|
| s11, s22, s12, dspss | 4 | ✅ 推奨 | 応力・変位は IR で直接/間接測定 |
| temperature, thermal_smises | 2 | ✅ 推奨 | 温度は IR の直接測定量 |
| ux, uy, uz | 3 | △ 任意 | 変位計測がある場合 |
| x, y, z, 法線, 曲率 | — | ❌ 非推奨 | 幾何は測定ノイズの主因ではない |

## 2. ノイズモデル案

### 2.1 加法ガウスノイズ（基本）

```
x_noisy = x + ε,  ε ~ N(0, σ²)
```

- **σ の決め方**：  
  - 応力：`σ = k * std(stress)`（例：k = 0.02–0.05 → 2–5% 相当）  
  - 温度：絶対値の 1–3% 程度

### 2.2 乗法ノイズ（放射率むら相当）

```
x_noisy = x * (1 + η),  η ~ N(0, σ²)
```

- 局所的な較正誤差を表現
- σ ≈ 0.01–0.03 程度

### 2.3 混合モデル（推奨）

```
x_noisy = x * (1 + η) + ε
η ~ N(0, σ_mul²),  ε ~ N(0, σ_add²)
```

- 乗法：較正・放射率むら
- 加法：センサノイズ

### 2.4 実装場所

- **オプション A**：`prepare_ml_data.py` 内で、グラフ構築後に `data.x` の該当列にノイズ付加
- **オプション B**：`train.py` の DataLoader で、`transform` としてオンザフライで付加（推奨）
  - 毎エポックで異なるノイズ → 実質的なデータ拡張
  - 検証時はノイズなしで評価

## 3. クラス比率（欠陥あり vs 欠陥なし）

### 3.1 実運用での想定

構造健全性モニタリングでは、**欠陥なしが大多数**であることが一般的：

| 分野 | 想定比率（健全:欠陥） | 根拠 |
|------|----------------------|------|
| 航空機・宇宙構造 | 95:5 〜 99:1 | 定期検査で異常は稀 |
| インフラ | 90:10 〜 95:5 | 劣化は時間経過で増加 |
| 製造検査 | 80:20 〜 95:5 | 工程・製品による |

### 3.2 サンプルレベル vs ノードレベル

- **サンプルレベル**：1 サンプル = 1 フェアリング（欠陥あり or なし）
- **ノードレベル**：1 グラフ内で、欠陥ノード数 << 健全ノード数

現状：
- サンプル：欠陥あり 100、欠陥なし 1 → 約 99:1（欠陥あり優勢）
- ノード：1 サンプルあたり欠陥ノードは数十〜数百、健全は 1 万以上 → ノードはすでに不均衡

### 3.3 推奨サンプル比率

| 目的 | 健全:欠陥 | 根拠 |
|------|-----------|------|
| **実運用に近い評価** | 90:10 〜 95:5 | 実際の検査分布に合わせる |
| **学習の安定性** | 70:30 〜 80:20 | 欠陥サンプルが少なすぎると学習困難 |
| **バランス重視** | 50:50 | 検出性能の上限評価用 |

**推奨：80:20（健全 80%, 欠陥 20%）**

- 実運用寄り（健全優位）を保ちつつ、欠陥サンプルも十分確保
- ノードレベルの不均衡は Focal Loss で対応済み
- 必要に応じて 90:10 や 70:30 で感度分析可能

### 3.4 実現方法

現状：欠陥あり 100、欠陥なし 1

**案 1：欠陥なしのノイズ拡張で水増し**

- healthy_baseline 1 サンプルから、ノイズ付加で N バリアント生成
- 例：σ=0.03 で 50 バリアント → 健全 50、欠陥 100 → 約 33:67
- さらに増やすと 80:20 に近づけられる（例：80 バリアント → 80:100 ≒ 44:56）

**案 2：欠陥なしサンプルを FEM で追加生成**

- 同一ジオメトリで欠陥パラメータを 0 にしたケースを複数実行
- 負荷条件を少し変えるなどで多様性を持たせる
- 計算コストは増えるが、物理的に妥当

**案 3：ハイブリッド（推奨）**

- 欠陥なし 1 サンプルをノイズ拡張で 79 バリアント作成
- 健全 80、欠陥 100 → **約 44:56**（やや欠陥多め）
- または健全 100 バリアント → 100:100 で 50:50
- 80:20 に厳密に合わせるなら、欠陥サンプルを 20 にサブサンプルするか、健全を 400 バリアントにする必要あり

**実務的な落としどころ：**

- **健全:欠陥 = 1:1（50:50）**：  
  - 健全 1 をノイズ拡張で 99 バリアント → 100:100  
  - 実装が簡単で、検出性能のベースライン評価に適する

- **健全:欠陥 = 4:5（約 44:56）**：  
  - 健全 1 → 80 バリアント、欠陥 100 のまま  
  - 実運用よりやや欠陥多めだが、学習はしやすい

- **健全:欠陥 = 4:1（80:20）**：  
  - 健全 1 → 400 バリアント、欠陥 100  
  - 実運用に近いが、同一ベースの拡張が多くなり過学習リスクあり  
  - 複数条件の健全 FEM を追加する方が望ましい

## 4. 実装ロードマップ

| 段階 | 内容 | 優先度 |
|------|------|--------|
| 1 | ノイズ拡張（加法ガウス）を DataLoader transform で実装 | 高 |
| 2 | prepare_ml_data に healthy_baseline を含め、健全サンプルを認識 | 高 |
| 3 | 健全サンプルのノイズ拡張で複数バリアントを生成 | 高 |
| 4 | クラス比率オプション（50:50, 80:20 等）を追加 | 中 |
| 5 | 乗法ノイズ・混合ノイズのオプション追加 | 低 |

---

## 5. まとめ

| 項目 | 提案 |
|------|------|
| **ノイズ拡張** | 応力・温度特徴に加法ガウス（σ≈2–5%）を、学習時のみオンザフライで付加 |
| **クラス比率** | まず 50:50 で検証し、余裕があれば 80:20 を目標に健全サンプルを増やす |
| **実装** | `train.py` の DataLoader に `transform` を追加し、`prepare_ml_data` で healthy_baseline を組み込む |

この方針で、実測 IR に近い条件での学習と、実運用を意識したクラス比率の両立を図れる。
