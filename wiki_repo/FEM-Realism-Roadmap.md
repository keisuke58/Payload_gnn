## 2. 実装ロードマップ — 優先度順

### Tier 1: 最高優先 (欠陥検出の本質に直結)

#### 2.1 Cohesive 要素による接着層モデル化

| 項目 | 内容 |
|------|------|
| **現状** | Tie 拘束 (完全結合, 厚さゼロ) — 剥離は「ノード削除」で表現 |
| **目標** | Cohesive Zone Model (CZM) で界面応力・エネルギー解放率を物理的に再現 |
| **Abaqus 実装** | COH3D8 (六面体) or COH3D6 (楔形), Traction-Separation 則 |
| **効果** | 欠陥部の応力集中パターンが物理的に正しくなる → GNN 検出精度に直結 |
| **難易度** | ★★★★☆ |
| **推定工数** | 2–3 ヶ月 (パラメータ調整含む) |
| **再現度向上** | 10% → **70%** (接着物理) |

**必要な材料パラメータ** (CFRP/エポキシ接着剤の典型値):

| パラメータ | 記号 | 典型値 | 備考 |
|-----------|------|--------|------|
| 初期剛性 (法線) | K_n | 10⁵–10⁶ N/mm³ | 接着層厚 ~0.2mm から算出 |
| 初期剛性 (せん断) | K_s | 10⁴–10⁵ N/mm³ | |
| 法線強度 | t_n | 30–60 MPa | 接着剤の引張強度 |
| せん断強度 | t_s | 25–50 MPa | |
| Mode I 破壊エネルギー | G_Ic | 0.2–0.5 N/mm | DCB 試験から |
| Mode II 破壊エネルギー | G_IIc | 0.5–1.5 N/mm | ENF 試験から |

**Abaqus 実装イメージ**:

```python
# Cohesive セクション定義
model.CohesiveSection(name='Sec-Adhesive',
    material='Mat-Adhesive',
    response=TRACTION_SEPARATION,
    initialThicknessType=SPECIFY,
    initialThickness=0.2)

# Traction-Separation 則
mat.MaxsDamageInitiation(table=((t_n, t_s, t_s),))
mat.DamageEvolution(type=ENERGY,
    mixedModeBehavior=BK, power=2.284,
    table=((G_Ic, G_IIc, G_IIc),))
```

**欠陥挿入方法の変更**:
- 現行: Tie 拘束からノードを除外 → 人工的な不連続
- CZM: 接着層の初期損傷を D=1.0 に設定 → 物理的に整合した剥離

## 5. 最大のゲームチェンジャー: Cohesive Zone Model

### 5.1 なぜ Cohesive が最重要か

現行の「Tie 拘束 + ノード除去」方式の問題:

```
現行 (Tie):
  健全部: スキン ──Tie──> コア  (完全結合, 変位一致)
  欠陥部: スキン ──────> コア  (Tie なし, 完全自由)
  → 欠陥境界で「0 or 1」の不連続 → 非物理的な応力特異点

Cohesive:
  健全部: スキン ──[Cohesive D=0]──> コア  (弾性接着)
  欠陥部: スキン ──[Cohesive D=1]──> コア  (完全損傷, 面圧のみ)
  遷移部: スキン ──[Cohesive 0<D<1]──> コア  (部分損傷)
  → 欠陥境界で連続的な応力遷移 → 物理的に正しい応力集中パターン
```

### 5.2 GNN 学習への直接的効果

| 特徴量 | Tie 方式 | Cohesive 方式 |
|--------|---------|--------------|
| 変位 (U) | 欠陥部で自由膨張 (過大) | 面圧で制約された現実的な膨張 |
| 応力 (S) | 欠陥境界で特異点 (非物理的) | 欠陥先端で連続的な応力集中 |
| ひずみ (ε) | 不連続ジャンプ | 滑らかな勾配 |
| 界面トラクション | 取得不可 | **法線/せん断トラクション** (新特徴量候補) |
| エネルギー解放率 | 取得不可 | **SERR (G_I, G_II)** (進展予測に直結) |

→ 34 次元 + 界面トラクション/SERR で **最大 38 次元** に拡張可能

### 5.3 実装戦略

```
Phase A (2週間): 単一要素の CZM 検証
  - 1要素 DCB/ENF テスト → 荷重-変位曲線が文献値と一致するか確認
  - パラメータ感度: K_n, t_n, G_Ic の影響を把握

Phase B (4週間): フェアリングモデルへの統合
  - InnerSkin-Core / OuterSkin-Core 界面に薄層 Cohesive メッシュを挿入
  - メッシュ生成: Sweep で 1 要素厚のコヒーシブ層
  - 既存 Tie を Cohesive セクションに置換

Phase C (2週間): 欠陥挿入方式の変更
  - 欠陥領域の Cohesive 要素に初期損傷 (D=1.0) を SDVINI で付与
  - 部分損傷 (0<D<1) による「成長中」欠陥のモデル化

Phase D (4週間): パラメータ校正 + データ生成テスト
  - 健全モデル: 界面トラクション分布の確認
  - 欠陥モデル: 変位・応力の妥当性検証
  - extract_odb_results.py の拡張 (SDEG, CSTRESS 出力追加)
```

## 6. 実現可能性の正直な評価

### 2年で全部いけるか？

| 項目 | 判定 | 根拠 |
|------|:----:|------|
| ダブラー | ✅ 確実 | 数日の作業 |
| 面内異方性 | ✅ 確実 | Abaqus CompositeLayup で対応済み基盤あり |
| 対称BC | ✅ 確実 | 数行の追加 |
| Cp(z) 空力圧力 | ✅ 確実 | AnalyticalField で 1 週間 |
| **Cohesive 接着層** | ✅ 可能 | Abaqus CZM は成熟機能。**最重要タスク** |
| オジーブ補剛 | ✅ 可能 | ShellExtrude + Tie。ジオメトリ干渉の調整が主 |
| 過渡熱解析 | ✅ 可能 | HeatTransfer ステップの追加 |
| 製造残留応力 | ✅ 可能 | ΔT 初期条件のみ |
| 音響 (簡易) | ✅ 可能 | 擬似静圧として |
| **分離衝撃** | ⚠️ 挑戦的 | Explicit ソルバー, 率速度依存材料, 火工品モデル。到達 50–60% |

### ボトルネック

1. **Cohesive メッシュの計算コスト**: CZM 要素追加で DOF が 1.5–2 倍に増加 → frontale (93GB) でも厳しくなる可能性 → marinos (メモリ確認要) or メッシュ最適化
2. **Abaqus 生成速度**: 1 サンプル 5 分 → 5,000 サンプルで 17 日 (連続稼働) → FNO サロゲートが鍵
3. **パラメータ同定**: Cohesive のパラメータは文献値の範囲が広い → 実験データ (JAXA 共同研究) が望ましい

### 最終到達見込み

```
分離衝撃を除く全項目実装:  ~85–88%
分離衝撃も含めて:          ~90%  (Explicit は挑戦的)
```

**85–90% は十分に論文で "high-fidelity FEM" と主張できるレベル**。
残り 10–15% は実機固有の詳細 (ボルト孔形状, 実測材料定数, 動的効果) で、シミュレーション研究では通常モデル化しない領域。
