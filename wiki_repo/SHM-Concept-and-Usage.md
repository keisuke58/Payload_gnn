# SHM システムの概念と運用シナリオ

> 本ドキュメントは、H3フェアリング向けGNN-SHMシステムの技術的位置づけと、地上試験から飛行中までの運用シナリオを定義します。

## 1. 結論：打ち上げ前の検査に使えるか？

**はい、非常に有効です。**
本SHMシステムは、飛行中（オンライン）だけでなく、**打ち上げ前の地上検査（オフライン）** においても、従来の検査手法を代替・補完する強力なツールとなります。

### 地上での利用メリット
1.  **アクセス性**: フェアリング内部や高所など、検査員がアクセスしにくい場所でも、センサさえ貼ってあればコネクタ接続（または無線）だけで検査可能です。足場を組む必要がありません。
2.  **高速化・自動化**: 超音波探傷（UT）のようにプローブを全面走査する必要がなく、電気的なスイッチ操作だけで構造全体の健全性を数秒で診断できます。
3.  **ベースライン取得**: 「健全な状態」のデータを地上で取得しておくことで、飛行中に発生した変化（損傷）をより高感度・正確に検知するための基準（Reference）となります。

## 2. 運用ライフサイクルとSHMの役割

本システムは、製造から打ち上げ終了まで、フェアリングのライフサイクル全体を通じて活用されます。

### Phase 1: 地上検査 (Pre-launch)

このフェーズでは、**「製造品質の保証」** と **「打ち上げ準備状態の確認」** が主目的です。

| タイミング | 用途 | 従来手法との比較 |
| :--- | :--- | :--- |
| **製造直後** | **初期欠陥スクリーニング**<br>製造時の接着不良（ディスボンド）、層間剥離、ボイドの検出。 | **UT/X線の代替**<br>全面スキャンにかかる膨大な工数を削減。怪しい箇所だけを従来のNDIで精密検査する「スクリーニング」として機能。 |
| **輸送後** | **輸送ダメージ確認**<br>工場→射場への輸送中に受けた衝撃や振動による損傷（BVID: 目に見えない衝撃損傷）の有無を確認。 | **目視点検の限界突破**<br>カバーや梱包を解かずに、内部の健全性を確認可能。 |
| **射場組立時** | **組立・結合検査**<br>衛星フェアリング結合時のミスアライメントや過度な締結による損傷監視。 | **リアルタイム監視**<br>作業中に常時モニタリングすることで、事故を未然に防ぐ。 |
| **発射台 (On-pad)** | **最終 Go/No-Go 判断**<br>アクセス不可能な状態で、打ち上げ直前の健全性を最終確認。 | **唯一の手段**<br>アクセスアームが外れた後でも、電気的にチェックが可能。 |

### Phase 2: 飛行中監視 (In-flight / Launch)

このフェーズでは、**「No-Growth（欠陥が進展していないこと）の実証」** が主目的です。

*   **リアルタイムモニタリング**: 音響振動（Max Q）や分離衝撃イベント時に、既存の欠陥が拡大していないか監視します。
*   **イベント検知**: デブリ衝突などの突発事象を検知します。
*   **データ間引き**: 機上のエッジAIが推論し、地上へは「健全/異常あり」のステータスのみを送信します。

## 3. 技術的背景：従来NDIとの比較

構造ヘルスモニタリング（SHM）は、従来の非破壊検査（NDI）を進化させた概念です。

**非破壊評価 (NDE) の体系:**
```mermaid
graph LR
    NDE[非破壊評価] --> NDI[従来のNDI/NDT]
    NDE --> SHM[構造ヘルスモニタリング]
    NDI --> Off[定期点検<br>オフライン]
    SHM --> On[常時監視<br>オンライン]
    SHM --> Dig[デジタルツイン<br>(本研究: FEM+ML)]
    SHM --> Sen[センサ実測<br>(将来: ガイド波)]
```

| 項目 | 従来 NDI | 本研究 (GNN-SHM) |
| :--- | :--- | :--- |
| **タイミング** | 定期点検時（オフライン） | **常時 (地上〜飛行中)** |
| **手法** | 超音波探傷、X線、打音検査 | FEMシミュレーション + GNN推論 |
| **人手** | 熟練検査員が必要 | **自動化・自律化** |
| **対象** | 検査可能な箇所のみ（局所） | **構造全面（グローバル）** |
| **目的** | 欠陥の「有無」判定 | 欠陥の **位置・サイズ・種別** まで特定 |

## 4. なぜ「デジタルツイン」なのか？

現在（Year 1）はFEMデータを用いていますが、これは以下の理由によります。

1.  **学習データの確保**: AIの学習には大量の「欠陥データ」が必要です。実機を壊してデータを取ることは不可能なため、FEMで数千パターンの欠陥を模擬し、デジタル空間で学習させます。
2.  **物理的解釈性**: GNNを用いることで、センサデータと構造の幾何学的情報（メッシュ）を紐づけ、どの場所がどう壊れているかを物理的に解釈しやすくします。

将来的（Year 2）には、このFEMで鍛えたモデルを、少量の実測データで調整（ドメイン適応）し、実機に適用します。
